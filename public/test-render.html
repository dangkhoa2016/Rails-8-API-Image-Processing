Image Render Smoke Test

This page uses Vue 3 CDN, so no build tool is required. The flow is: sign in to get a JWT, enter an image URL, then load each rendered variant in sequence to inspect performance, rendering failures, and preview results from the API.

Recommended test flow

After generating the test list, run only the scenario groups you actually need. This avoids firing every request at once, reduces the risk of hitting the rate limit, and saves resources when a full sweep is unnecessary.

1

Sign in

The JWT is read from the Authorization header returned by /users/sign_in.

Open this page from ./test-render.html when possible so it shares the same origin as the API. If you open it elsewhere, the Authorization header must be exposed through CORS.

Enter image URL

The render list is available only after a successful sign-in.

Each card below shows the full API GET URL. Images are fetched with fetch plus the Bearer token because <img src> does not attach the Authorization header automatically.

On public domains, the default is a same-origin static image to avoid hotlink rate limits. If you use hosts such as Wikimedia, some providers may return 429 after a few consecutive requests. On localhost, same-origin images are not usable because the SSRF guard blocks loopback requests.
